## 0. Принципы UX

Каждое решение ниже опирается на конкретную боль или персону из предыдущих документов, а не на общие best practices "для красоты":

| Принцип | Почему | Источник |
|---|---|---|
| **Постепенное доверие** — банк не обязателен для старта | Недоверие к передаче банковских данных | PRD §3.10 |
| **Скорость ввода превыше всего** | Приложение забрасывают на 2–3 неделе, если ввод трения | PRD §3.5, персоны 3/8/5 |
| **Простой режим по умолчанию, глубина — по запросу** | Низкая финграмотность (41,5%), пугают термины YNAB-типа | PRD §3.6, персона 6/9 |
| **Не осуждающий тон в состояниях "плохо"** | Стыд/тревога у пользователей в долговой ловушке | Персона 14 |
| **Полная функциональность офлайн/при слабом интернете** | Нестабильный интернет в регионах | Персона 13 |
| **Двуязычность RU/KZ как first-class, не как перевод строк** | Гигиенический минимум локального рынка (Mewnay) — важно не уступить даже в этом | Market Research §5 |
| **Free-тариф не должен ощущаться "урезанным намеренно"** | Виральность и миссия финграмотности требуют щедрого Free | PRD §7 |
| **Крупные target-зоны и минимум шагов** | Персона 6 (пенсионер), 13 (регион) — низкая цифровая грамотность | Персоны 6/13 |

## 1. Навигация

### Структура (Information Architecture)

Нижняя навигация из 5 зон с приподнятой центральной кнопкой быстрого добавления (паттерн, привычный по Kaspi.kz и большинству финтех-приложений в регионе — снижает когнитивную нагрузку на новых пользователей):

```
┌──────┬──────┬───────┬──────┬──────┐
│Главная│Кошелёк│  (+)  │  AI  │ Ещё  │
│  🏠  │  💳  │  ⊕   │  🤖  │  ⋯   │
└──────┴──────┴───────┴──────┴──────┘
```

- **Главная** — Dashboard
- **Кошелёк** — счета/кошельки + история транзакций
- **(+)** — не экран, а модальный Bottom Sheet быстрого добавления транзакции (см. §15), всегда на расстоянии одного тапа из любого места приложения
- **AI** — чат с AI-ассистентом (отдельная вкладка, а не спрятано в "Ещё" — это центральный дифференциатор продукта)
- **Ещё** — хаб второстепенных, но не менее важных разделов: Статистика, Цели, Инвестиции, Подписки, Настройки

### Почему не 8 вкладок вместо хаба "Ещё"

8 экранов верхнего уровня (Dashboard, Wallet, Statistics, AI, Goals, Investments, Subscriptions, Settings) не помещаются в нижнюю навигацию без потери читаемости. Раздел **"Ещё"** — не свалка, а структурированный список с иконками и коротким описанием, отсортированный по частоте использования персоны (см. §10).

### Паттерны навигации внутри разделов

- **Свайп-вкладки** внутри разделов с несколькими под-видами (например, Кошелёк: "Все счета" / отдельный счёт)
- **Back-жест / кнопка "Назад"** всегда возвращает на предыдущий экран стека, а не на Dashboard — сохраняет ментальную модель
- **Deep links** от push-уведомлений и AI-инсайтов ведут в контекст (конкретная транзакция/бюджет), но back-стек всегда достраивается до Dashboard, чтобы пользователь не терялся

### Accessibility и локализация навигации

- Подписи вкладок видны всегда (не только иконки) — критично для персон с низкой цифровой грамотностью (6, 9, 13)
- При включённом крупном системном шрифте иконки не сжимаются — подписи переходят на 2 строки, а не обрезаются

## 2. Splash

**Назначение:** мгновенный бренд-момент на холодном старте, пока приложение проверяет сессию и грузит критичный минимум данных.

```
┌─────────────────────────┐
│                         │
│                         │
│        [ЛОГОТИП]        │
│       AI Finance        │
│                         │
│      • • •  (loader)    │
└─────────────────────────┘
```

- **Длительность:** жёсткий таргет ≤ 1200 мс восприятия (реальная загрузка может занимать дольше, но splash не должен "залипать" дольше — при долгой загрузке плавно переходит в skeleton-состояние Dashboard, см. §13, а не держит пользователя на статичном логотипе)
- **Никаких интерактивных элементов** — это единственный экран без единого таппабл-элемента в приложении
- **Ветвление после Splash:** есть валидная сессия → Dashboard (с skeleton-состоянием пока грузятся данные); нет сессии → Onboarding
- **Офлайн:** если при старте нет сети и нет закэшированной сессии — сразу переход на экран [Offline] (§14), а не бесконечный loader

## 3. Onboarding

Экраны и порядок подробно описаны в [04_User_Flows.md §2–4](04_User_Flows.md#2-первый-запуск-и-onboarding) — здесь фокус на UX-решениях, не дублирующих user flow.

### Ключевые UX-решения

- **3 value-экрана, не больше.** Тестирование конкурентов (Copilot, Monarch) показывает, что >4 экранов онбординга роняют конверсию до регистрации у финтех-аудитории с низкой мотивацией ждать.
- **"Продолжить как гость"** видна на первом же экране выбора авторизации, а не спрятана мелким текстом внизу — это прямая реализация принципа "постепенное доверие": пользователь может пощупать ввод транзакции до того, как отдаст номер телефона.
- **Прогресс онбординга** отображается не точками "1 из 5", а смысловыми чекпоинтами ("Язык → Кто вы → Первая трата"), чтобы пользователь понимал *зачем* каждый шаг, а не просто сколько осталось.
- **Экран "Зачем вам AI Finance?"** (выбор целей) — мультивыбор карточек, не радио-кнопки: у персоны "Молодая семья" одновременно верны цели "бюджет с семьёй" и "накопить на цель".
- **Первая транзакция гарантированно происходит до конца онбординга.** Если пользователь пропускает подключение источников, экран 4.3 из User Flow ведёт не на пустой Dashboard, а сразу предлагает добавить первую трату (фото чека доступно прямо из окна "Настрою позже") — момент первой ценности ("aha moment") не должен зависеть от банковской интеграции.

### Состояние ошибки OTP

- После 3 неудачных попыток — не блокировка, а мягкая подсказка "Код не пришёл?" с опцией повторной отправки и альтернативным каналом (звонок с озвучиванием кода) — важно для персон с нестабильным интернетом (13) и мигрантов (11), у которых SMS иногда задерживается на роуминге.

## 4. Dashboard

**Назначение:** ответить на главный вопрос пользователя за 2 секунды без единого тапа: "Сколько у меня денег и что происходит с моим бюджетом сейчас?"

```
┌─────────────────────────────┐
│ 👋 Привет, Данияр      🔔 ⚙️│
├─────────────────────────────┤
│  Остаток на месяц            │
│  245 300 ₸  из 380 000 ₸     │
│  ▓▓▓▓▓▓▓▓░░░░░░░░  64%       │
├─────────────────────────────┤
│ 🤖 AI-инсайт                 │
│ "Траты на такси выросли      │
│  на 20% за неделю"    →      │
├─────────────────────────────┤
│ 📅 Ближайший платёж           │
│ Kaspi Рассрочка · 15 000 ₸    │
│ через 2 дня              →   │
├─────────────────────────────┤
│ Бюджеты                  Все →│
│ 🍔 Еда      ▓▓▓▓▓░░ 70%       │
│ 🚕 Транспорт▓▓▓▓▓▓▓ 95% ⚠️    │
├─────────────────────────────┤
│ Последние операции        Все →│
│ Magnum        -12 400 ₸       │
│ Kaspi перевод  +50 000 ₸      │
└─────────────────────────────┘
```

### Иерархия зон (сверху вниз = по убыванию важности)

1. **Приветствие + быстрый доступ** к уведомлениям и настройкам
2. **Главная метрика** — остаток бюджета периода (не "баланс всех счетов" — это частая ошибка конкурентов: пользователю важнее "сколько можно ещё потратить", чем абстрактная сумма на счетах)
3. **Карточка AI-инсайта** — ровно одна, самая релевантная на сегодня (не лента из 5 карточек — это создаёт баннерную слепоту). Если релевантных инсайтов нет — карточка заменяется мягким CTA ("Спросите AI о своих финансах")
4. **Ближайший платёж по рассрочке** — показывается только при наличии активных рассрочек (💎); один из немногих виджетов, ради которого стоит рассмотреть Premium прямо с Dashboard
5. **Бюджеты** — превью 2–3 самых "горячих" категорий (близких к лимиту), не все сразу
6. **Лента операций** — последние 3–5, с явной ссылкой "Все" в Кошелёк

### Взаимодействия

- **Pull-to-refresh** обновляет курсы валют и пересчитывает виджеты
- **Тап по любому виджету** ведёт вглубь соответствующего раздела (см. диаграмму в User Flows §5)
- **Долгий тап на остатке бюджета** переключает вид "на месяц / на неделю" — учитывает персон с нерегулярным доходом (3, 8), которым месячный горизонт не подходит

### Состояния

- **Skeleton-загрузка**, **Empty (новый пользователь)**, **Offline**, **Ошибка** — см. общие паттерны §11–14, применённые к Dashboard: в частности, Empty-Dashboard заменяет виджет бюджетов на CTA "Создать первый бюджет", а ленту операций — на CTA "Добавить первую операцию".

## 5. Wallet (Кошелёк)

**Назначение:** единая точка правды о том, где физически лежат деньги — счета, карты, наличные, мультивалютные кошельки — и полная история операций.

### Структура

- **Верхняя карусель счетов** — горизонтальный скролл карточек: "Наличные", "Kaspi Gold", "Halyk Card", "USD-кошелёк" 💎 и т.д. Каждая карточка — баланс + иконка банка/типа. Карточка "+ Добавить счёт" всегда последняя в карусели.
- **Суммарный net worth** под каруселью — сумма по всем счетам, пересчитанная в базовую валюту по актуальному курсу (для мультивалютных пользователей — 💎, с указанием "по курсу на сегодня, ₸/$")
- **Единая лента транзакций** ниже, с фильтром по счёту (тап на карточку в карусели фильтрует ленту, а не открывает отдельный экран — меньше переходов)
- **Свайп по строке транзакции**: влево — удалить, вправо — быстро сменить категорию (частый паттерн исправления неверной AI-категоризации)

### Экран одного счёта (детали)

- Баланс, мини-график динамики за период, кнопки "Пополнить/списать вручную" (для наличных и счетов без синхронизации), "Импортировать выписку" 🤖, "Архивировать счёт"

### Добавление счёта — Bottom Sheet (см. §15)

Три варианта одним списком: "Наличные" (мгновенно, без полей кроме названия) / "Импортировать выписку банка" 🤖 / "Мультивалютный кошелёк" 💎 (выбор валюты из списка мировых)

### Особые состояния

- **Расхождение баланса** (пользователь считает, что на счёте больше/меньше денег, чем показывает импорт): банер "Сверить баланс?" с возможностью ручной корректирующей операции "Корректировка" — без этого пользователи теряют доверие к точности приложения при первом же расхождении.

## 6. Statistics (Статистика/Отчёты)

**Назначение:** дать персонам с аналитическим складом (IT-специалист, Инвестор) глубину, а всем остальным — один понятный график без необходимости "разбираться".

### Структура

- **Переключатель периода** сверху (Неделя/Месяц/Год/Произвольный — последнее 💎)
- **Главный график** — круговая диаграмма по категориям 🆓 с легендой (тап по сектору = фильтр ленты снизу)
- **Вторичные вкладки-свайпы**: "По категориям" 🆓 / "Тренды" 💎 (линейный график динамики категории во времени) / "Heat map" 💎 (календарная тепловая карта дней с высокими тратами — прямое попадание в боль "не понимаю в какие дни утекают деньги")
- **Сравнение периодов** ("в этом месяце на 15% больше, чем в прошлом") — текстовая плашка над графиком, не отдельный график — так быстрее считывается персонами без аналитического бэкграунда
- **Кнопка "Обсудить с AI"** внизу каждого отчёта — прямой мост в AI Chat с уже переданным контекстом текущего графика
- **Экспорт** 💎 (PDF/Excel) — иконка в правом верхнем углу, не отдельный пункт меню

### UX-примечание

Ни один график не показывается без подписи-вывода в одну строку (например: "Транспорт — самая быстрорастущая категория"). Персоны с низкой финграмотностью считывают текст быстрее, чем интерпретируют график самостоятельно — график дополняет вывод, а не заменяет его.

## 7. AI Chat

Технический поток и точки входа — в [04_User_Flows.md §10](04_User_Flows.md#10-ai-ассистент). Здесь — детальная UX-проработка самого экрана.

```
┌─────────────────────────────┐
│  ← AI-ассистент              │
├─────────────────────────────┤
│                 Вы: Сколько  │
│                 потратил на  │
│                 такси в июне?│
│                               │
│ 🤖 За июнь вы потратили       │
│ 32 400 ₸ на такси — это на    │
│ 20% больше, чем в мае.        │
│ Основная причина — поездки    │
│ вечером в будни.               │
│ [Показать операции →]         │
│                               │
├─────────────────────────────┤
│ 💡 Сократить траты на такси   │
│ 💡 Как накопить на цель       │
├─────────────────────────────┤
│ 🎙  Напишите вопрос...    ➤  │
└─────────────────────────────┘
```

### Ключевые UX-решения

- **Чипы с предложенными вопросами** над полем ввода меняются контекстно (на основе последнего ответа AI и текущих данных пользователя) — снижают "паралич пустого поля", особенно важный барьер для персон с низкой финграмотностью, которые не знают, *что* спросить.
- **Ссылки-переходы внутри ответа** ("Показать операции →", "Открыть цель →") — AI-ответ должен быть точкой навигации, а не тупиковым текстом.
- **Голосовой ввод вопроса** — та же иконка микрофона, что и при добавлении транзакции (консистентность паттерна Whisper-ввода во всём приложении).
- **Индикатор "печатает..."** с анимацией, а не спиннер — снижает ощущение "зависания" при задержке ответа модели (500мс–3с).
- **Тон ответов** — обязательно не осуждающий (см. принцип §0), особенно для инсайтов о перерасходе или рассрочках: "Вы близки к лимиту" вместо "Вы превышаете бюджет".
- **История чата сохраняется** и доступна свайпом вниз как обычная переписка — пользователь может вернуться к прошлым инсайтам.
- **Ограничение Free-тарифа** реализовано не через блокировку экрана целиком, а через лимит сообщений в день с мягким счётчиком ("Осталось 2 вопроса сегодня") и ненавязчивым CTA на Paywall при исчерпании — резкая блокировка чата воспринимается враждебно к продукту, который позиционируется как "помощник".

## 8. Goals (Цели накоплений)

### Список целей

- Карточки в виде "банок с деньгами" (jar metaphor) с наглядным прогресс-баром и процентом — метафора накопления интуитивно понятна без обучения (важно для персон 6, 9, 13)
- Для целей в иностранной валюте 💎 — маленький бейдж валюты и подпись "с учётом курса"

### Детали цели

- Прогресс-бар крупно вверху, дата достижения при текущем темпе накопления (AI-прогноз 🤖: "При таком темпе вы достигнете цели к марту 2027")
- Кнопка "Отложить сейчас" — быстрый перевод суммы в цель без похода в Кошелёк
- Опция "Откладывать автоматически X% с каждого дохода" — переключатель, не отдельная сложная настройка

### Создание цели (Bottom Sheet, см. §15)

Минимум полей на первом экране (название + сумма + срок), валюта и авто-накопление — под "Показать больше настроек" — не перегружаем первый шаг созданием цели.

### Особый UX-момент для персоны "Пара перед крупной покупкой"

Цель может иметь **со-владельца** — второй участник видит тот же прогресс-бар в своём приложении, оба получают уведомления о вкладах друг друга. Это единственная точка пересечения Goals и семейного/шеринг-функционала (§9 в User Flows), поэтому она обозначена явно, чтобы не потерялась при разработке.

## 9. Investments (Инвестиции)

**Назначение:** закрыть боль персоны "Инвестор" (разрозненные активы, нет net worth) и частично IT-специалиста — без превращения приложения в брокерский терминал.

### Структура

- **Общий Net Worth** сверху — сумма всех активов (депозиты, недвижимость, акции, крипта, золото) минус обязательства (кредиты/рассрочки из §8 User Flows) — это единственное место в приложении, где долги и активы показаны в одном уравнении
- **Разбивка по классам активов** — горизонтальные полосы (не пироги — для 4+ классов активов горизонтальные полосы читаются точнее)
- **Карточки отдельных активов** (вручную вводятся или импортируются): "Депозит Halyk — 2 400 000 ₸", "Квартира (оценка) — 35 000 000 ₸", "Акции — $3 200"
- **Реальная доходность** 🤖 — показатель с поправкой на инфляцию/девальвацию тенге, с коротким пояснением, откуда цифра взялась (доверие к непрозрачному расчёту — частый источник недоверия у финансово продвинутых пользователей)

### UX-примечание про сложность

Раздел по умолчанию свёрнут в упрощённый вид ("У вас есть накопления и активы на сумму X") для персон, которым это не близко, и полностью раскрывается для персон 4/7 через переключатель "Подробный вид" — тот же принцип "простой режим по умолчанию, глубина по запросу" из §0, применённый локально.

## 10. Subscriptions (Подписки и регулярные платежи)

> Уточнение области: это экран **обнаруженных и отслеживаемых регулярных платежей пользователя** (Netflix, спортзал, мобильная связь) — AI-функция из PRD §6 ("обнаружение подписок и утечек"), а не управление подпиской самого приложения AI Finance (это Paywall/Settings, см. §12 User Flows).

### Структура

- **Список обнаруженных подписок** 🤖 — карточка на каждую: название мерчанта, сумма, периодичность, дата следующего списания, тренд цены ("Цена выросла на 15% с марта" — прямое попадание в боль "подписочный крип")
- **Суммарная строка** сверху: "Регулярные платежи: 18 400 ₸/мес" — агрегированная цифра, которую пользователь обычно никогда не считал сам
- **Действие по карточке**: "Это не подписка" (обучает модель категоризации) / "Напомнить перед списанием" / "Хочу отменить" (не выполняет отмену за пользователя — юридически и технически не наша зона ответственности, но даёт короткую инструкцию, как отменить именно этого провайдера)

### Пустое состояние

Если подписок не обнаружено (мало истории транзакций) — не пустой экран, а объяснение: "Мы ищем повторяющиеся платежи. Как только наберётся 2–3 месяца истории, здесь появятся находки" — управляет ожиданиями, а не выглядит как баг.

## 11. Settings (Настройки)

### Структура (список секций)

1. **Профиль** — имя, телефон/email, аватар
2. **Тариф** — текущий статус (Free/Premium), дата продления, ссылка на Paywall
3. **Локализация** — язык (RU/KZ/EN), основная валюта, формат даты/чисел
4. **Счета и семейный доступ** — управление кошельками, приглашение участников
5. **Уведомления** — гранулярные переключатели (лимиты бюджета, платежи по рассрочкам, AI-инсайты, маркетинг) — не единый тумблер "все уведомления", чтобы пользователь не отключал полезные вместе с шумом
6. **Безопасность** — Face ID/биометрия, PIN-код, список активных сессий/устройств
7. **Данные** — экспорт всех данных, удаление аккаунта (с двойным подтверждением и объяснением последствий)
8. **Поддержка** — чат поддержки, FAQ, обучающие материалы по финграмотности (партнёрский контент, см. PRD §6)
9. **О приложении** — версия, политика конфиденциальности, условия использования

### UX-примечание

Удаление аккаунта и отключение уведомлений о лимитах бюджета **не должны** быть скрыты глубже одного тапа "нарочно" — тёмные паттерны прямо противоречат принципу доверия из §0 и миссии продукта.

## 12. Empty States

Единый паттерн для всех разделов: **иллюстрация + заголовок с сутью + один явный CTA**, без текста длиннее двух строк.

| Экран | Заголовок | CTA |
|---|---|---|
| Dashboard (новый пользователь) | "Добавьте первую операцию — и мы покажем, куда уходят деньги" | Добавить операцию |
| Кошелёк | "Здесь появятся ваши счета" | Добавить счёт |
| Статистика | "Нужно немного данных, чтобы построить первый график" | Добавить операцию |
| Цели | "Пока нет целей — накопите на что-то важное" | Создать цель |
| Инвестиции | "Добавьте свои накопления и активы, чтобы увидеть Net Worth" | Добавить актив |
| Подписки | см. §10 — специальное объясняющее состояние, а не CTA | — |
| AI Chat | "Спросите что угодно о своих финансах" + 3 примера вопросов | Тап по примеру = отправка |
| Поиск/фильтр без результатов | "Ничего не найдено по «{запрос}»" | Сбросить фильтр |

**Принцип:** пустое состояние — это не "недостаток данных", а первый шаг обучения продукту. Никогда не показывать пустой экран без объяснения и действия.

## 13. Offline

**Философия:** приложение должно оставаться полезным без интернета — это прямое требование персоны "Житель региона" (13) и общая ситуация с мобильным интернетом вне крупных городов Казахстана.

### Что работает офлайн

- Просмотр уже загруженных данных (Dashboard, Кошелёк, Статистика — из локального кэша)
- **Добавление транзакций вручную и через OCR** — сохраняются локально в очередь синхронизации, UI показывает маленький индикатор "не синхронизировано" (иконка облака с точкой) на таких операциях
- Просмотр целей, бюджетов, рассрочек (последнее известное состояние)

### Что недоступно офлайн

- AI Chat (требует backend) — при попытке открыть без сети показывается баннер "AI-ассистент недоступен офлайн" вместо ошибки/пустого чата
- Импорт выписок, синхронизация счетов, оплата Premium

### UI-паттерн

Тонкий неблокирующий баннер сверху экрана ("Нет подключения — изменения сохранятся локально"), а не модальное окно — модалка прерывала бы работу с уже начатым вводом транзакции. При восстановлении сети — баннер на 2 секунды меняется на "Синхронизировано" и исчезает.

---

## 14. Loading

### Принцип: skeleton screens, не спиннеры

Для Dashboard, Кошелька, Статистики — контентные заглушки (серые прямоугольники по форме будущего контента), сохраняющие layout, чтобы не было "прыжка" интерфейса при подгрузке данных.

### Спиннеры допустимы только для:

- Коротких точечных действий внутри уже загруженного экрана (сохранение транзакции, отправка сообщения AI)
- OCR-обработки фото чека — с текстовой подписью "Распознаём чек…" (пользователь должен понимать, что именно происходит, а не просто видеть крутилку)

### Долгая загрузка (>5 сек)

Появляется дополнительная строка-подсказка ("Это займёт чуть больше времени, чем обычно") — снижает тревогу от неопределённости, не позволяя пользователю решить, что приложение зависло.

## 15. Ошибки

### Классификация и тон

| Тип ошибки | Пример | Тон и решение |
|---|---|---|
| Сетевая ошибка | Не удалось отправить запрос к AI | Нейтрально-техническое: "Не получилось получить ответ. Повторить?" + кнопка retry |
| Ошибка ввода пользователя | Некорректная сумма/дата | Инлайн под полем, конкретно: "Сумма должна быть больше 0" |
| Финансовая "плохая новость" (не техническая ошибка) | Превышение бюджета, риск просрочки платежа | **Не в стиле системной ошибки** — не красный крестик, а спокойная предупреждающая карточка с советом от AI, без осуждающих формулировок (см. §0) |
| Критическая ошибка (потеря данных, оплата не прошла) | Сбой синхронизации, неудачный платёж Premium | Модальное окно с понятным объяснением на человеческом языке (никогда не показывать сырой код ошибки/stack trace пользователю) и явным следующим шагом ("Написать в поддержку") |
| OCR/AI не смог распознать | Чек нечитаем, голос не распознан | Не ошибка, а приглашение к действию: "Не удалось прочитать чек — введите сумму вручную" с уже открытой формой ручного ввода |

### Общий принцип

Финансовое приложение работает с тревожной для многих персон темой (долги, нехватка денег) — техническая ошибка и "плохая финансовая новость" **всегда визуально различимы**: первая красная/системная, вторая — спокойная, в фирменных тонах, с фокусом на решение, а не на проблему.

## 16. Bottom Sheets

Основной паттерн для второстепенных действий, не заслуживающих полноценного экрана — сохраняет контекст (пользователь видит, откуда он пришёл, полупрозрачный фон предыдущего экрана).

### Где используются

- Быстрое добавление транзакции (центральная кнопка навигации, §1)
- Выбор способа ввода (вручную/фото/голос/шаблон)
- Добавление счёта в Кошельке
- Создание цели/бюджета (первый шаг)
- Приглашение участника в семейный бюджет
- Быстрая смена категории транзакции по свайпу
- Confirm-действия среднего риска (архивировать счёт, отменить цель) — но **не** для необратимых действий (удаление аккаунта — только полноэкранный modal с явным вторым подтверждением)

### Правила поведения

- Открывается снизу с системной анимацией, занимает от 40% до 90% высоты экрана в зависимости от контента (никогда не на весь экран — иначе это уже полноценный экран, а не bottom sheet)
- Закрывается свайпом вниз, тапом по затемнённой области или явной кнопкой — все три способа работают всегда
- Если пользователь начал ввод данных внутри шита (например, вводит сумму транзакции) — свайп для закрытия требует подтверждения ("Отменить ввод?"), чтобы не терять данные случайным жестом
- Внутри одного bottom sheet **не бывает** вложенной навигации на 2+ уровня — если нужно углубиться дальше, это повод открыть полноценный экран
