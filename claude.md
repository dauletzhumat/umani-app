Правила

Перед написанием любого кода

1. Сначала составить план
2. Получить подтверждение
3. После реализации написать тесты
4. После тестов обновить документацию

Никогда не писать большие изменения за один раз.

Каждая задача должна занимать максимум один рабочий день.

Всегда соблюдать единый стиль кода.
